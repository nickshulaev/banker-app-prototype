# AML-опросник — полная спецификация (22 вопроса, 6 секций)

Источник: `Опросник AML (1).pdf`. Это под-флоу шага `aml_form` ветки **eResidency** в онбординге.
Реализовать **целиком** — все 22 вопроса. Структура ниже — готовый конфиг: перенести в
`onboarding.jsx` как массив `AML_QUESTIONS`, рендер — обобщённый по `type`.

## Поведение под-флоу
- 6 секций (экранов-групп), сверху прогресс-точки (6 шагов): «Банковские операции», «Открытие
  счёта», «Открытие счёта», «Работа и образование», «Финансы», «Дополнительно».
- Внутри секции вопросы идут подряд; кнопка низа: «Следующий вопрос» / на последней секции —
  «Завершить».
- Назад — к предыдущей секции. По «Завершить» → следующий шаг ветки (`address`).
- Ответы складываются в `formData.aml[questionId]`. Валидацию делать мягкой (прототип): не блокируем,
  но required-вопросы подсвечиваем.

## Типы полей (`type`)
`single` — один вариант · `multi` — несколько · `text` — строка · `number` — число (USD) ·
`country` — выбор страны (одна) · `countryMulti` — несколько стран · `file` — прикрепить документ.
Поле `conditional` — показывать дочернее поле при значении родителя.

## Рендер (обобщённо)
Один компонент `AmlQuestion({ q, value, onChange, C })` ветвится по `q.type`:
- `single`/`multi` — список карточек-вариантов (чек/радио), стиль — как существующие списки выбора
  в аппе; `multi` — мультивыбор. `otherInput` — текстовое поле при выборе «Другое».
- `text`/`number` — поле по образцу `FormField`.
- `country`/`countryMulti` — строка-селектор со стрелкой (можно простой моковый список стран).
- `file` — кнопка «Прикрепить документ» (моковая, без реальной загрузки).
- `conditional.field` — показывать под вопросом, когда выбран `when`.
- `collapsible` (Q13) — показать первые N вариантов + «Показать ещё».

## Acceptance (дополнение к основному ТЗ)
- Анкета содержит **все 22 вопроса**, сгруппированы по 6 секциям с прогресс-точками.
- Условные поля работают: Q02→файл, Q08→банки, Q16→стоимость недвижимости, Q01/Q13 «Другое»→текст.
- Q13 «Показать ещё» раскрывает полный список сфер.
- «Завершить» на 6-й секции ведёт к шагу `address`.
